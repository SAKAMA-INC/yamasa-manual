1

概要・前提条件

概要

サカマアプリ（yamasaApp）は招待制のB2Bマーケットプレイスです。新規ユーザーの登録には、事前に招待コード（= Firestore の Company ドキュメントID）を発行する必要があります。

前提条件

  • ウルアプリ（app-post-ios / 販売者向けアプリ）がインストール済みで、ログインできること
  • 対象の取引先ユーザーが既にウルアプリの取引先一覧に登録されていること

登録の全体フロー

1 招待コードの取得（ウルアプリから）
↓
2 ユーザーへ招待コードを送付
↓
3 ユーザーがアプリで新規登録
↓
4 登録後の確認
↓
5 必要に応じて追加設定
2

招待コードの取得（ウルアプリから）

招待コードは、ウルアプリ（販売者向けアプリ）のマイページから取得できます。

操作手順

  1. ウルアプリを起動してログイン
  2. 下部タブから「マイページ」を開く
  3. 「招待コード取得」セクションの「取引先一覧」をタップ
  4. ユーザー一覧が表示される（検索バーでユーザー名検索も可能）
  5. 対象の取引先ユーザーをタップ
  6. 「{ユーザー名}様の招待コードをコピーしました。」とアラートが表示される
  7. クリップボードに招待コードがコピーされる
コピーされる招待コードは Firestore の Company ドキュメントID（version/1/companies/{companyDocument}）です。

クリップボードには以下の形式でコピーされます：

{ユーザー名}様のヤマサアプリの招待コードはこちら {companyDocument（招待コード）}

Firebase Console から直接作成する場合（取引先が未登録の場合）

取引先一覧に対象ユーザーがいない場合は、Firebase Console で手動作成も可能です。

  1. Firebase Console にログイン
  2. 対象プロジェクトを選択
  3. 左メニュー「Firestore Database」を開く
  4. version > 1 > companies コレクションに移動
  5. 「ドキュメントを追加」をクリック
  6. 以下のフィールドを設定
  7. 「保存」をクリック
  8. 生成されたドキュメントIDをメモ — これが招待コード
フィールド名型値備考
ドキュメントID-自動ID or 任意の文字列これが招待コードになる
namestring会社名（例: 〇〇商店）必須
createdAttimestamp現在日時必須
updatedAttimestamp現在日時必須
3

ユーザーへの招待コード送付

以下の情報をユーザーに伝達します：

  • 招待コード: 手順1で作成したドキュメントID
  • アプリのダウンロードリンク（App Store / Firebase App Distribution）
アプリ内にも招待コードリクエスト機能があり、ユーザーが s.shibata@yamasa.tokyo 宛にメールを送信できます。
4

ユーザー側の登録操作（アプリ内）

ユーザーが以下の操作を行います。

登録手順

  1. アプリを起動
  2. トップ画面で「はじめての方」をタップ
  3. 登録画面で以下の情報を入力
  4. プライバシーポリシーに同意
  5. 「はじめる」をタップ
入力項目必須説明
ユーザー名はい店舗名・会社名（例: 〇〇商店）
メールアドレスはいログインに使用
パスワードはい8文字以上推奨
招待コードはい手順1で発行したコード

登録時にシステムが自動で行う処理

1 招待コード検証（Company ドキュメント存在確認）
↓
2 Firebase Auth ユーザー作成（email/password）
↓
3 以下を並列実行:
• Company ドキュメント作成
• CompanyUser ドキュメント作成
• companyJoin API 呼び出し（招待元の会社にユーザーを紐付け）
• User データ取得・キャッシュ
• CompanyUser データ取得・キャッシュ
↓
4 メイン画面へ遷移
5

登録後の確認

Firebase Console で以下を確認します。

Authentication の確認

  1. Authentication > 「Users」タブを開く
  2. 新しいユーザーのメールアドレスと UID が表示されることを確認

Firestore companyUsers の確認

version/1/companyUsers/{firebaseUid} を開き、以下のフィールドが正しく設定されていることを確認します。

フィールド確認内容
firebaseUidFirebase Auth の UID と一致
companyDocument招待コードの会社ドキュメントIDと一致
email登録したメールアドレス
name登録したユーザー名
isAdmin未設定（null）

Firestore users の確認

version/1/users/ 配下で、companyDocument フィールドから該当会社のユーザーを検索して確認します。

6

登録後の追加設定（必要に応じて）

管理者権限の付与

CompanyUser の isAdmin フラグを true に設定します。

  1. Firestore > version/1/companyUsers/{firebaseUid} を開く
  2. isAdmin フィールドを追加（boolean: true）

ユーザー詳細情報の設定

version/1/users/{firebaseUid} で以下を設定可能です。

フィールド型説明
userTypenumber0: アプリ事前決済 / 1: アプリ事後決済 / 2: Pro事後決済
postCodestring郵便番号
citystring都市
townstring町域
areastringエリア
telstring電話番号
mobilestring携帯番号
shopCategorystring店舗カテゴリ
customerCodestring顧客コード
attributeTypestring「個人」or「飲食店もしくは法人」
paySitestring支払い条件
payMethodnumber0: 振込 / 1: カード
7

既存会社への追加ユーザー登録

同じ会社に複数ユーザーを追加する場合の手順です。

  1. 同じ招待コードを新しいユーザーに渡す
  2. ユーザーがアプリで通常の登録手順を実施
  3. companyJoin API により同じ会社に自動的に紐付けされる
既存の招待コードをそのまま使い回すだけでOKです。追加のFirestore操作は不要です。
8

トラブルシューティング

🔑 「入力された招待コードは見つかりませんでした。」
招待コード（Company ドキュメントID）が Firestore に存在しない
Firebase Console で version/1/companies/ 配下に該当ドキュメントが存在するか確認
name フィールドが空
Company ドキュメントの name が設定されているか確認
📧 「メールアドレスは既に使用されています」
同じメールアドレスで Firebase Auth に既にユーザーが存在
Firebase Console > Authentication で該当ユーザーを確認。必要に応じてパスワードリセットまたはユーザー削除
📦 登録完了後にデータが表示されない
Firestore への書き込みが部分的に失敗した可能性
version/1/companyUsers/{uid} が存在するか確認
companyDocument フィールドが不正
companyDocument フィールドが正しい招待コードと一致しているか確認
users データが欠落
version/1/users/ 配下に該当ユーザーのデータがあるか確認
🔒 ログインできない
Firebase Auth のユーザーが存在するが Firestore データが不整合
Firebase Auth でユーザーの存在を確認し、version/1/companyUsers/{uid} のデータを確認。必要に応じて CompanyUser ドキュメントを手動作成
📂
